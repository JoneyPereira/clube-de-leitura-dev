# 📘 Capítulo XI — Conclusão

O último capítulo do livro não traz novos conceitos técnicos, mas sim algo ainda mais importante:

> **uma mentalidade sobre como evoluir no design de software.**

---

# 🧠 Ideia central

> Você **não cria um bom design de primeira**.

Isso é fundamental.

Projetar bem não é:

* acertar tudo na primeira tentativa
* prever todos os cenários
* criar uma arquitetura perfeita

Projetar bem é:

✔ iterar
✔ refatorar
✔ aprender continuamente

---

# 🔄 Design é um processo contínuo

O capítulo reforça que o design de software deve ser:

* evolutivo
* incremental
* guiado por feedback

Ou seja:

```id="v39z0l"
Escreve código → Aprende → Refatora → Melhora o design
```

---

# 🧪 Aprendizado com o próprio código

Uma ideia muito forte aqui:

> O código que você escreve hoje te ensina como melhorar amanhã.

Isso conecta com tudo que vimos no livro:

* SOLID
* Encapsulamento
* LSP, OCP, ISP
* Design Smells
* Métricas

Todos esses conceitos são **ferramentas para melhorar o código ao longo do tempo**.

---

# 🔧 Refatoração como prática constante

O livro reforça algo essencial:

> Refatorar não é opcional — é parte do desenvolvimento.

Sem refatoração:

* o código degrada
* os smells aumentam
* o sistema fica difícil de evoluir

Com refatoração contínua:

* o design melhora
* o sistema se mantém saudável
* mudanças ficam mais seguras

# 📚 Leitura contínua (Próximos passos)

O autor recomenda alguns livros clássicos para aprofundamento:

---

## 📖 Agile Principles, Patterns, and Practices

Autor: Robert C. Martin

* aprofunda princípios do SOLID
* conecta design com práticas ágeis

---

## 📖 Growing Object-Oriented Software, Guided by Tests

Autores:

* Steve Freeman

* Nat Pryce

* foco em design guiado por testes (TDD)

* mostra evolução de sistemas reais

---

## 📖 Design Patterns

Autores: Gang of Four

* catálogo clássico de padrões de projeto
* base para muitas soluções OO

---

## 📖 Refactoring to Patterns

Autor: Joshua Kerievsky

* conecta refatoração com design patterns
* mostra como evoluir código real

# 🧭 Mentalidade final do livro

Depois de todos os capítulos, a mensagem final é:

> Não existe código perfeito. Existe código **em evolução**.

---

# 📌 Resumo final do livro

Ao longo do livro aprendemos:

* escrever código mais coeso
* reduzir acoplamento
* usar abstrações corretamente
* evitar design smells
* aplicar princípios SOLID
* pensar em arquitetura

Mas o mais importante:

> **Aprender a pensar sobre o design do código.**
